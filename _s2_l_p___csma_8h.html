Detailed Description

Configuration and management of S2-LP CSMA.

Author
LowPower RF BU - AMG
Version
1.2.1
Date
16-April-2018

The S2LP CSMA feature, when configured and enabled, is transparent for the user. It means the user has only to call the S2LPCsmaInit() function on a filled structure and then enable the CSMA policy using the S2LPCsma() function.

Example:

CsmaInit csmaInit={
S_DISABLE, // persistent mode
TBIT_TIME_64, // Tbit time
TCCA_TIME_3, // Tcca time
5, // max number of backoffs
0xFA21, // BU counter seed
32 // CU prescaler
};
...
S2LPCsmaInit(&csmaInit);
S2LPCsma(S_ENABLE);
Note
The CS status depends of the RSSI threshold set. Please see the S2LP_Qi module for details.
